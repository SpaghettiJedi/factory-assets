### Decisions not in the spec

### Be aware

**Blocking sleep during retry**: The daemon blocks for the full 30s between checkout attempts using `Atomics.wait`. This is intentional (simpler than async sleep, appropriate for the single-threaded daemon), but means no other poll work happens during that window. Each target's checkout retry pauses that target's processing only, not the entire pass.

**Evidence only on rescue, not on final failure**: The `checkout_retry` evidence event fires only when the retry succeeds. If both attempts fail, the operation fails normally with no special evidence marker. This matches the WorkOrder's "when the retry rescues the pass" language.
